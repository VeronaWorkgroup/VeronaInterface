This document defines the communication between a **Testcenter** and a **Player** to execute computer based assesments as offered for the German [VERA - Vergleichsarbeiten](https://www.iqb.hu-berlin.de/vera), or the [National Assessment Studies and IQB Trends in Student Achievement](https://www.iqb.hu-berlin.de/bt). A **Player** in this context is a HTML page or fragment possibly containing a javascript applicaton to present and handle interaction with an assessment unit. An **Testcenter** is piece of software that handles user authentication, unit selection, sequence and user input data, in short the management of an assessment.
The defintion was proposed and is currently in discussion.
## Introduction The **Player** is currently expected to be loaded into an [`iframe`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/iframe#Attributes) in the **Testcenter** using the `srcdoc` attribute. Both ends of this interface communicate via [`postMessage`](https://developer.mozilla.org/en-US/docs/Web/API/Window/x-doc-messaging). The only data saved beyond the lifetime of a **Player** is the data transfered in `restorePoint`.

## Communication squence ### General course After the **Player** loads it announces it's state by a `ReadyNotification` which is answered by the **Testcenter** with an `DataTransfer` including the `unitDefinition` containing the unit and all subunits supposed to be presented. Upon start the **Player** announces the beginning of the assesment of the unit with a `StartedNotification`. Entries submitted by the assessee are transfered to the **Testcenter** via a `ChangedDataTransfer`. ### Navigation If the current page is supposed to change due to events from the **Testcenter** this is communicated via a `NavigateToPage` message. If the event to change the page is originated in the **Player** it sends a `PageNavigationRequest` or a `UnitNavigationRequest` respectively, if the loaded unit should be changed.

NavigateToPage Testcenter ðŸ¡’ Player

Requests the player to navigate to the given page. Completion is announced by the **Player** via `ChangedDataTransfer`. ```typescript { type: 'vo.ToPlayer.NavigateToPage'; sessionId: string; newPage: string; } ``` * newPage specifies to which page to navigate and is one of the strings given via `validPages`
